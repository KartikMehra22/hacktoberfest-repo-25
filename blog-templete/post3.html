Core Web Vitals in 2025: what actually matters

CLS, INP, and LCP explained with practical fixes you can ship today.

By Alex Kumar Jul 5, 2025 Web Performance 6 min read

Understanding Core Web Vitals

As of 2025, Google measures three key metrics that directly influence user experience and search ranking: Largest Contentful Paint (LCP), Interaction to Next Paint (INP), and Cumulative Layout Shift (CLS). These metrics collectively define how fast, stable, and responsive your site feels.

LCP: Speed of meaningful paint

LCP tracks how long it takes for the largest visual element (like a hero image or heading) to render. Aim for < 2.5s. Compress images, use CDN caching, and reduce render-blocking JS to improve it.

INP: The new responsiveness metric

INP replaces FID in 2024 and captures the delay between user input and visual response. Poor INP often points to heavy JavaScript or unoptimized event handlers. Break large tasks, use requestIdleCallback, and prefer async rendering.

CLS: Visual stability matters

CLS measures how much elements shift unexpectedly as the page loads. To fix it, always define width/height attributes for images, and avoid injecting content above existing elements without proper spacing.

Practical fixes

  • Use font-display: swap; to avoid layout shifts from web fonts.
  • Lazy-load below-the-fold media but preconnect to critical origins.
  • Run Lighthouse CI or WebPageTest regularly to monitor regressions.

Quick example

<img 
  src="hero.jpg" 
  width="1200" 
  height="600" 
  loading="lazy" 
  alt="Hero banner"
/>
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
